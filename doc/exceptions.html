Exception Handling in the CruiseControl.NET Server

This document outlines the likely causes of exceptions that can occur within CruiseControl.NET and how the CruiseControl.NET server attempts to handle these exceptions when they are raised. There are two broad scenarios for the generation of exceptions: during initialisation and during operation.

Initialisation Phase Exception Handling

Initialisation exceptions occur when the server is launched. These exceptions are typically related to invalid server or remoting configuration data. Initialisation exceptions will cause the server to shut down immediately and report the exception to the console for the command-line runner and to the system event log for the Windows Service runner.
