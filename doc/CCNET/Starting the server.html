CruiseControl.NET : Starting the server
This page last changed on Apr 13, 2004 by orogers.

You can launch the CruiseControl.NET server by executing the ccnet.exe file. This file will be located in the \server folder if you have downloaded the CCNet zip distribution or in \build\server if you have downloaded the source and are running the NAnt build.

Executing the server using the -help flag (eg. ccnet.exe -help) will display a list of the command-line arguments accepted by the server. They are: 
Options:
  -config:[ccnet.config]
  -remoting:[on/off] default:off
  -project:[projectName]
  -help

Executing the ccnet.exe file without any command-line arguments will cause the server to look for the configuration file ccnet.config in the current directory. If this file does not exist, the server will report: "Specified configuration file does not exist: ccnet.config". If you have multiple configuration files or if your configuration file has a different name, you should use the -config: flag to specify the name of the configuration file to use.

By default, the CCNet server will start in standalone mode. If you want to be able to access the server using CCTray or using the WebDashboard, you will need to instruct the server to launch as a remotable server by using the -remoting:on flag.

There is a StartCCNet.bat file that can be used to launch the server using a particular set of flags. You may choose to customise this batch file to launch the server using your selected set of flags.

When setting up the CruiseControl.NET server and debug configuration problems, it is often useful to force the build for a particular project (this enables you to run the full build even if there are no new modifications). Launching the server using the -project: flag will force a build for the specified project.
