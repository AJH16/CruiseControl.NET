CruiseControl.NET : Web Dashboard Reporting Features
This page last changed on Nov 01, 2004 by mroberts.

This is preliminary documentation for new features in the Web Dashboard to provide reporting and configuration features.

These new features are still in development and should not be considered to be 'production ready'. Some reporting features (e.g. FxCop reporting) have not yet been included in the new reporting pages, but these will be added soon. Use these new features at your own risk!  

The new features in the Web Dashboard consolidate the previous, separate, Web and Web Dashboard applications, plus also will provide the following new features:

  • Multi-project, multi-server support. Now you just need 1 deployment of the web application to report over all your CCNet projects. (This code is complete, but some reporting features have not yet been migrated)
  • Project Configuration. Add, edit and delete projects through the web application on any CCNet build server (remote or local to the web application) (This code is still in early days. Only some scenarios using Perforce, File Source Control, Command Line Builder and NAnt Builder have been implemented)

The features are being used on CCNetLive at http://ccnetlive.thoughtworks.com/ccnet/

If you do want to use the new web application, please note the SECURITY section below.

The 'plugins' API for the new web application is completely different to that for the old web application. Documentation on writing your own plugins will come later.

60 second setup guide

To setup the new Web Dashboard features, just run through the instructions on the Web Dashboard page.

Security and the CCNet web application

Certain features within the CCNet web application could allow a user to seriously damage the setup of the machine that a build server is hosted on. Unless you trust everyone with access to your deployed web application, do the following:
  1. Open up the web.config file in the root directory of your deployed web application
  2. Look for a line that says <add key="AllowSecureActions" value="true" />
  3. Delete this line, comment it, or change the value setting to false
  4. Save the file

This process will disable all actions that could compromise your build servers, including all configuration features.

Usage guide

This part of the documentation is very sparse at the moment, and some of the instructions here will change once more features are added (and bugs are fixed!)

At the moment, you need to update the <webURL> tag for each project in your build server (this will be updated soon). The format of the URL should be the following:

  • <webURL>http://hostname/VirtualDirectory/Controller.aspx?_action_ViewProjectReport=true&server=Server Name&project=Project Name</webURL>
  • Server Name is the name of the server that you set in the 'web.config' file for the web application. If you didn't set anything, it will just be local
  • Project Name is the same as the name that you set for the project in the config file.
  • In both of the above, make sure to replace any spaces in server and project names with %20.

Viewing projects

Assuming you've set everything up OK, you should be able to view the default page in the Virtual Directory you setup for your web application. The default page on CCNetLive is at http://ccnetlive.thoughtworks.com/ccnet/, and you should see something similar.

What you are seeing here is the farm view. There are 4 different views in the CCNet Web Application, with increasingly narrower scope. Here's a brief description:

  • Farm - shows all projects running on all servers. Drill down into a project by clicking on its name
  • Server - shows all projects running on a server. From here you can also view the 'server log', and add projects to a server
  • Project - Shows a summary of the current state of a summary, shows the recent builds, and allows project configuration. Drill down into a specific build by clicking on its name
  • Build - This is the equivalent of the old CCNet web application. Here you can view the results of a build in various ways (summary report, full log, FxCop results, etc.)

Note the location bar near the top of the page - this allows you to back-track up through the views.

Project configuration

Project configuration support is in very early days - we recommend you don't use it now and come back to it later. If you do have any questions about the project configuration features, please mail the ccnet-devel list.
