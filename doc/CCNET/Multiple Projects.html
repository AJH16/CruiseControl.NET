CruiseControl.NET : Multiple Projects
This page last changed on Apr 21, 2004 by orogers.

The CruiseControl.NET server can be configured to manage the integration requirements for several projects concurrently. To do this, simply include the configuration for the different projects consecutively in the ccnet.config file.

<cruisecontrol>
     <project name="project1">
          <state type="state" filename="project1.ccnet.state" />
          ...
     </project>
     <project name="project2">
          <state type="state" filename="project2.ccnet.state" />
          ...
     </project>
</cruisecontrol>

One important caveat is to specify different filenames in the <state> element for each project. If this is not specified then both projects will use the same state file ("ccnet.state") which can lead to some pretty confusing results.

Similarly, it is important to use different log folders for the <xmllogger> element. Otherwise, CCNet will have no way of determining which builds originated from which projects.

At this point, the CCNet web application only supports displaying the results for a single project. So you will need to install and configure one CCNet web application per project.
