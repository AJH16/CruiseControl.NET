CruiseControl.NET : Commit Process
This page last changed on Apr 20, 2006 by orogers.

Please adhere to the following process when commiting changes to the repository:

  1. check to make sure that the build is currently passing on ccnetlive.
  2. validate that your changes work by running 'b.bat' in the root folder
  3. create a new issue on jira describing the change or locating an existing issue
  4. commit your changes, including the jira issue number at the start (ie. "CCNET-142: my comment...")
  5. monitor the build on ccnetlive and verify it completes successfully
  6. if it is complete, close the jira issue and include the successful build label in the comment.
