CruiseControl.NET : ProjectExceptions
This page last changed on Apr 20, 2004 by orogers.

If you are experiencing periodic failures in your build process (such as sporadic VSS or CVS connection problems), it is possible to configure the CCNet server to not publish the exception as a failed build. Instead the exception will simply get logged to the ccnet.log file. This means that the team will not be confused by builds that seem to be failing for inexplicable reasons. To do this, you will need to add the publishExceptions attribute to the <project> element in your ccnet.config file and set it to false.

<cruisecontrol>
    <project name="proj" publishExceptions="false">
        ...
    </project>
</cruisecontrol>
