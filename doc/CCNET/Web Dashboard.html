CruiseControl.NET : Web Dashboard
This page last changed on Nov 01, 2004 by mroberts.

The CCNet Web Dashboard Application is used for reporting summary details for a set of servers and projects. To install it:

  1. Unzip the distribution as described in Basic Installation
  2. In IIS, map a virtual directory to the 'webdashboard' subfolder of your CCNet folder

The Dashboard needs to connect over the network to each of Servers you want to report on. If you have changed any of the remoting configuration for your servers, you'll need those details now. (See Server Application Config File for more details on configuring remoting.)

To configure the dashboard, in the <CCNET> \ <servers> section of the dashboard's 'web.config' file, add a <server> tag for each CruiseControl.NET Server you want to monitor. Set the url attribute to the management URL for the Server. Set the name attribute to be a unique name for each server you configure in the config file.

The web.config file that ships with the dashboard contains an example that works for a default scenario.

From version 0.7, the Web Dashboard includes preliminary reporting features. These are not yet stable, and the recommended reporting application is still the Project Report Web Application. If you want to use the new Dashboard features though, read up on the Web Dashboard Reporting Features.
