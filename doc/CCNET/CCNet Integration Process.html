CruiseControl.NET : CCNet Integration Process
This page last changed on May 30, 2004 by orogers.

The CruiseControl.NET server goes through the following steps when running an integration on your project:

  1. CCNet checks the source code repository for modifications that have occurred since the last build attempt. This check occurs periodically based upon the <schedule> element in ccnet.config. The <sourcecontrol> section determines which projects to check.
    1. When modifications are detected
      1. CCNet labels the project with CCNETUNVERIFIEDxxx (xxx = date/time) (VSS only)
      2. CCNet starts the process defined by the <build type="nant"> element in ccnet.config.
  2. NAnt processes the build script specified by the <build> element in ccnet.config. The following steps make up one possible scenario.
    1. The NAnt build script uses the <vssget> task to retrieve the projects to be built.
    2. NAnt runs VS.NET using either the <exec> task or the <solution> task. The <solution> task provides better reporting capability for later steps.
    3. NAnt runs NUnit using the <nunit2> task
    4. NAnt runs FxCop using the <exec> task
  3. CCNet receives the outcome of the NAnt execution (Success/Failed)
    1. If the build was successful, CCNet replaces the CCNETUNVERIFIEDxxx label with the next sequential build number. (VSS only)
    2. The XML output files are gathered from each step using the MergeFileTask and output as an xml file in the log directory specified in the <logDir> element.
  4. CCNet sends out email based on the <publishers><email> elements in ccnet.config (Optional).
    1. If <group notification="change"> is set, then the individuals will only receive email if the status of the build changes (Success -> Fail, or Fail -> Success)
    2. If <group notification="always"> is set, then the individuals will receive email for every build.
  5. CCTray displays current status of the build in the client system tray. The user can use the "Launch web page" menu item to browse to the build status website.
  6. Project Dashboard enables multiple continuous integration projects to be viewed.
