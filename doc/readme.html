CruiseControl.NET installation instructions

Preconditions

These instructions assume that you have already:
  • Installed Microsoft IIS
  • Installed the Microsoft .NET SDK (IIS must be installed before .NET)
  • Set up a NAnt build file to build and test your application
  • Placed your project under version control using CVS or VSS

Installing And Running the Server

  1. Start the installation by running the CruiseControl.NET Server Installer: ccnet.server.msi
  2. Select into which folder to install the CruiseControl.NET server. This folder will be referred to as the installation-folder.
  3. Select whether you would like to install CVS and NAnt. The NAnt install is strongly recommended because CruiseControl.NET needs to use enhancements only available in recent versions of NAnt. Installing CruiseControl.NET will not affect your existing installations of CVS or NAnt.
  4. Once the installation is complete, you must manually edit certain files to customize CruiseControl.NET to build your project. Edit the installation-folder\ccnet.config file. (eg. C:\Program Files\CruiseControl.Net\bin).
    1. Change the name attribute in the project tag to the name of your project.
    2. Change the projectUrl tag to point to the URL where you will install the CruiseControl.NET web application. For example http://localhost/ccnet.
    3. If you are using CVS, make the following changes to the CVS source control tag. Change the executable tag to point to cvs.exe. If you chose to install CVS during the installation you will not need to modify this value. Change the workingDirectory tag to point to the root folder containing the checked-out source for your project.
    4. If you are using VSS, first delete the HTML comment around the VSS source control tag and delete the CVS source control tag. Make the following changes to the VSS source control tag. Set the executable tag to point to location of the VSS ss.exe file. Set the project tag to the name of the VSS project that you want to monitor. Finally, set the VSS user name and password in the respective username and password tags.
    5. In the email section, change the user tags to map source control user names to valid email addresses. By default, a user in the developers group receives notification for all his or her changes, for all changes which break a build, and for all changes which fix a build while the buildmaster group receives notification for all changes.
  5. Edit the cruise.build file.
    1. If you are using CVS, modify the program attribute in the exec task to perform an update on your source tree. By default, this is set to the path where CVS is installed by CruiseControl.NET. Modify the basedir attribute to point to the root folder where the source will be checked out to.
    2. If using VSS, then delete the CVS exec task and remove the HTML comments around the VSS exec task. Change the program attribute to the location of the ss.exe file. Set the basedir attribute to point to the root folder where the source will be checked out to. Also, modify the commandline attribute to the include the name your VSS project.
    3. Modify the buildfile attribute in the nant task to point to the NAnt buildfile for your project.
  6. Assuming that everything is configured correctly, you should now be able to run CruiseControl.NET. Click the Run CruiseControl.NET icon in the under the Start menu->Programs->CruiseControl.NET.

Installing the Web Application

  1. Start the installation by running the CruiseControl.NET Web Installer: ccnet.web.msi
  2. Specify the virtual directory into which to install the CruiseControl.NET Web Application. This virtual directory must match the project URL specified in the Server installation above.
  3. Specify the folder where CruiseControl.NET will generate its log files. Typically this should be set to installation-folder\log. The specified folder must exist, otherwise the installation will not complete.
  4. Verify that the installation succeeded by browsing to the installed web application. For example http://localhost/ccnet.
